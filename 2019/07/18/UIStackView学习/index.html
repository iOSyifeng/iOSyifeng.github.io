UIStackView学习

发表于 2019-07-18

UIStackView官网译

UIStackView

用行或者列的方式布局视图集合的流线型接口

Declaration

Overview

Stack View 允许你利用自动布局的功能, 创建能够动态适应设备方向和可用空间的任何更改的用户界面. stack view 通过它的 arrangedSubviews属性管理者所有视图布局. 这些视图根据他们在arranged Subviews数组中的顺序沿着stack view的轴线排列. 确切的布局依赖于stack view的axis,distribution,slignment,spacing和其他属性

要使用一个stack view,打开一个你要编辑的故事版, 从对象库中拖出水平stack view或垂直stack view,并将stack view放到所需位置. 接下来,拖动stack的内容,将视图或者控件放入stack. 你也可以根据需要继续向stack view中添加视图和控件. 接口生成器会根据stack view的内容调整stack view的大小. 你也可以通过修改属性检查器中stack view的属性来调整stack view的外观

Note

你负责定义stack view的位置和大小(可选), 然后stack view管理其内容的布局和大小

Stack View and Auto Layout

stack view使用自动布局来定位和调整其排列好的视图的大小。stack view将第一个和最后一个排列好的视图的边缘沿堆栈轴对齐。在水平堆栈中，这意味着第一个排列视图的前缘固定在堆栈的前缘上，最后一个排列视图的后缘固定在堆栈的后缘上。在垂直堆栈中，顶部和底部边缘分别固定在堆栈的顶部和底部边缘上。如果将stack view的isLayoutMarginsRelativeArrangement属性设置为true，则stack view将其内容固定在相关的边距上，而不是边缘。

Positioning and Sizing the Stack View

尽管stack view允许您在不直接使用自动布局的情况下布局其内容，但仍然需要使用自动布局来定位stack view本身。通常，这意味着将stack view的至少两条相邻边缘固定在一起，以定义其位置。在没有附加约束的情况下，系统根据stack view的内容计算stack view的大小。

  • 沿着stack view的轴线，其拟合大小等于所有排列视图的大小加上视图之间的空间之和。

  • 垂直于stack view的轴线，其拟合大小等于最大排列视图的大小。

  • 如果stack view的isLayoutMarginsRelativeArrangement属性设置为true，则stack view的合适大小将增加，以包含空白。

您可以提供额外的约束来指定stack view的高度、宽度或两者兼备。在这些情况下，stack view调整其已安排视图的布局和大小，以填充指定区域。确切的布局取决于stack view的属性。看到UIStackView。分布和UIStackView。有关stack view如何处理其内容有额外空间或空间不足的完整描述的对齐枚举。

您还可以基于stack view的第一个或最后一个基线来定位stack view，而不是使用顶部、底部或中心Y位置。与stack view的拟合大小一样，这些基线是根据stack view的内容计算的。

  • 水平stack view返回forFirstBaselineLayout和forLastBaselineLayout方法的最高视图。如果最高的视图也是stack view，它将返回在嵌套stack view上调用forFirstBaselineLayout或forLastBaselineLayout的结果。

  • 垂直stack view为forFirstBaselineLayout返回其第一个已安排的视图，为forLastBaselineLayout返回其最后一个已安排的视图。如果其中一个视图也是stack view，那么它将返回在嵌套stack view上调用forFirstBaselineLayout或forLastBaselineLayout的结果。

请注意
基线对齐只适用于高度与其内部内容大小高度匹配的视图。如果视图被拉伸或压缩，基线将出现在错误的位置。

Managing the Stack View’s Appearance

UIStackView是UIView的一个非呈现子类;也就是说，它不提供自己的任何用户界面。相反，它只管理其已安排视图的位置和大小。因此，有些属性(如backgroundColor)对stack view没有影响。类似地，您不能覆盖layerClass、draw(:)或draw(:in:)。

有许多属性定义stack view如何布局其内容。

  • axis属性决定堆栈的垂直或水平方向。

  • distribution 属性决定了已排列视图沿堆栈轴线的布局。

  • alignment属性确定垂直于堆栈轴线的已排列视图的布局。

  • spacing 属性确定排列视图之间的最小间距。

  • isbaselinerativearrangement属性决定视图之间的垂直间距是否从基线测量。

  • isLayoutMarginsRelativeArrangement属性确定stack view是否将其排列好的视图相对于其布局边距进行布局。

通常，您使用一个stack view来列出少量项。您可以通过将stack view嵌套到其他stack view中来构建更复杂的视图层次结构。例如，图7显示了一个包含两个水平stack view的垂直stack view。每个水平stack view都包含一个标签和一个文本字段。

图7 嵌套stack view

您还可以通过向已安排视图添加附加约束来微调已安排视图的外观。例如，可以使用约束设置视图的最小或最大高度或宽度。或者可以为视图定义纵横比。stack view在布局其内容时使用这些约束。例如，在图像视图中有一个长宽比约束，该约束在图像调整大小时强制一个恒定的长宽比。

请注意
在向stack view内的视图添加约束时，要小心避免引入冲突。一般来说，如果视图的大小默认为给定维度的内部内容大小，则可以安全地为该维度添加约束。

Maintaining Consistency Between the Arranged Views and Subviews

stack视图确保它的arrangedSubviews属性始终是它的subviews属性的子集。具体来说，stack view执行以下规则:

  • 当stack view将视图添加到它的arrangedSubviews数组中时，它还将该视图作为子视图添加(如果还没有添加的话)。

  • 当从stack view中删除子视图时，stack view也将它从arrangedSubviews数组中删除。

  • 从arrangedSubviews数组中删除视图并不会将其作为子视图删除。stack view不再管理视图的大小和位置，但视图仍然是视图层次结构的一部分，如果它是可见的，则在屏幕上呈现。

尽管arrangedSubviews数组始终包含子视图数组的子集，但这些数组的顺序仍然是独立的。

  • arrangedSubviews数组的顺序定义了视图在堆栈中出现的顺序。对于水平堆栈，视图按读取顺序排列，较低的索引视图出现在较高索引视图之前。例如，在英语中，视图按从左到右的顺序排列。对于垂直堆栈，视图是从上到下排列的，索引视图越低，索引视图越高。

  • 子视图数组的顺序定义子视图的z顺序。如果视图重叠，索引较低的子视图将出现在索引较高的子视图后面。

Dynamically Changing the Stack View’s Content

每当视图被添加、删除或插入到arrangedSubviews数组中，或者当其中一个已安排的子视图的isHidden属性发生更改时，stack view自动更新其布局。

stack view还自动响应对其任何属性的更改。例如，可以通过更新stack view的axis属性动态更改堆栈的方向。

您可以将对已安排子视图的isHidden属性的更改和对stack view的属性的更改设置为动画，方法是将这些更改放置在动画块中。

最后，您可以在Interface Builder中直接为stack view的许多属性定义特定于大小类的值。每当stack view的size类发生更改时，系统自动激活这些更改。
